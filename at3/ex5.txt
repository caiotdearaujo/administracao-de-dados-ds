5. Descreva algumas diferenças e semelhanças entre planilhas eletrônicas, banco de dados, taxonomias, ontologias, tesauros, registros, data warehouse, data mart, data lake e data mesh?

- Planilhas eletrônicas: As planilhas eletrônicas são ferramentas que permitem armazenar e manipular dados tabulares. Elas são frequentemente utilizadas para cálculos simples, organização de dados e análise. Embora as planilhas sejam eficazes para análise e cálculos em pequena escala, elas podem ser limitadas em termos de capacidade de armazenamento, escalabilidade e colaboração quando comparadas a outras soluções mais robustas, como bancos de dados. Exemplos: Microsoft Excel, Google Sheets, Apple Numbers.
- Banco de dados: Bancos de dados também armazenam e gerenciam dados, mas de uma forma mais estruturada e escalável do que as planilhas eletrônicas. Os bancos de dados são projetados para lidar com grandes volumes de dados, oferecem funcionalidades avançadas de consulta e são capazes de garantir integridade, segurança e consistência dos dados. Exemplos: Oracle Database, Microsoft SQL Server, MySQL, PostgreSQL.
- Taxonomias, Ontologias e Tesauros: Todos esses termos se referem a estruturas de organização de informações, especialmente em contextos de categorização e classificação. Taxonomias: São sistemas de classificação hierárquica, geralmente usados para categorizar informações de forma ordenada. Ontologias: São estruturas que descrevem conceitos e suas relações de uma forma mais formal e semântica. Elas são usadas para representar conhecimento de domínio específico de uma maneira que seja compreensível para computadores. Tesauros: São listas de termos relacionados, geralmente usados para melhorar a precisão e a recuperação de informações em sistemas de busca. Exemplos: Taxonomias: Drupal Taxonomy, Wordpress Taxonomy; Ontologias: Protege, OntoStudio; Tesauros: AGROVOC, Medical Subject Headings (MeSH).
- Registros: Registros são unidades individuais de dados. Em contextos de gerenciamento de informações, registros podem se referir a entradas individuais em um banco de dados ou a documentos específicos em uma coleção. Exemplos: Sistemas de Gerenciamento de Banco de Dados (SGBD) como os mencionados anteriormente, pois eles tratam de armazenar registros de dados.
- Data Warehouse, Data Mart e Data Lake: Todos são ambientes para armazenamento e análise de dados, especialmente em grandes volumes. Data Warehouse: É um repositório centralizado que armazena dados de várias fontes em um formato estruturado e otimizado para consulta e análise. É comumente usado para suportar processos de tomada de decisão. Data Mart: É uma versão mais especializada e segmentada de um data warehouse, focada em um departamento específico ou em uma área de negócio. Data Lake: É um repositório de dados brutos e não estruturados que podem ser armazenados sem a necessidade de modelagem prévia. É flexível e escalável, permitindo o armazenamento de uma grande variedade de dados para análise posterior. Exemplos: Data Warehouse: Amazon Redshift, Snowflake, Google BigQuery; Data Mart: IBM Cognos, SAP BusinessObjects; Data Lake: Amazon S3, Azure Data Lake Storage, Google Cloud Storage.
- Data Mesh: Assim como os outros conceitos mencionados, o data mesh se relaciona ao gerenciamento e análise de dados em escala. O data mesh é uma abordagem arquitetônica mais recente, que propõe a descentralização do gerenciamento de dados, distribuindo a responsabilidade pela qualidade e governança dos dados entre equipes autônomas e interconectadas. Não há um software específico associado ao conceito de Data Mesh, pois ele é uma abordagem arquitetônica e cultural em vez de uma ferramenta específica. No entanto, tecnologias como Kubernetes, Apache Kafka, Apache Airflow e outras ferramentas de código aberto são comumente utilizadas em implementações de Data Mesh.
